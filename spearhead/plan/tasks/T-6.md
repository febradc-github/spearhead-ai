## Goal

Replace the T-1 stub with a working `search` MCP tool: the server starts
the watch pipeline on boot and returns ranked excerpts with source pointers.

## Expected files

- `mcp-server/server.js`
- `mcp-server/server.test.js`

## Depends on

T-1, T-5

## Acceptance criteria

- `server.js` starts the T-5 watch pipeline on boot.
- The `search` tool takes a natural-language query, embeds it, ranks index
  entries via `similarity.js`, and returns `{path, excerpt, score}` for the
  top results (default limit 8) (PROBLEM.md acceptance criterion 2).
- A missing API key or embeddings-call failure at query time surfaces as a
  clear, named tool error, not an empty/silent result (PROBLEM.md
  acceptance criterion 10).
- Tests exercise the tool handler against a fixture index (temp dir,
  pre-populated `embeddings.json`), asserting correct ranking and error
  surfacing; no live network calls.

## Out of scope

- Hook nudges (T-8, T-9).
- Naming/collision logic (T-7).

## Verification commands

- `node mcp-server/server.test.js`
