## Goal

Add the search-first directive to `remind.js`'s existing injected message
so agents are nudged to try the knowledge search tool before reading
source files directly.

## Expected files

- `hooks/remind.js`
- `rules/RULES.md`
- `hooks/remind.test.js`

## Depends on

none

## Acceptance criteria

- One line is added to `remind.js`'s injected message, present in both the
  full-rules refresh and the one-line anchor variants, directing the agent
  to try the knowledge-base search tool before reading source files
  (PROBLEM.md: search-first reminder).
- `rules/RULES.md` is updated with the same directive so `remind.js` stays
  byte-identical to it, per the existing test that enforces this.
- No change to `remind.js`'s cadence logic (session id resolution, idle
  expiry, refresh interval) — this task only changes injected text.

## Out of scope

- New hook files (T-9).
- Any MCP server or index logic.

## Verification commands

- `node hooks/remind.test.js`
