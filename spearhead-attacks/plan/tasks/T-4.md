## Goal

A single-file, atomically-written index store (`embeddings.json`) and an
in-process cosine-similarity ranking function.

## Expected files

- `mcp-server/lib/index-store.js`
- `mcp-server/lib/index-store.test.js`
- `mcp-server/lib/similarity.js`
- `mcp-server/lib/similarity.test.js`

## Depends on

T-1

## Acceptance criteria

- `index-store.js` reads/writes `spearhead-knowledge/index/embeddings.json`
  keyed by relative note path, each entry `{hash, embedding, updated,
  type}`; writes are atomic (temp file + rename), never leaving a
  partially-written file on disk (DESIGN.md ADR-002).
- `index-store.js` creates `spearhead-knowledge/index/` if it doesn't exist
  (no separate directory-setup task needed).
- `similarity.js` ranks index entries by cosine similarity against a query
  embedding, returning the top N with `{path, score}`.
- Tests use a temp directory (`node:fs.mkdtempSync`); no dependency on a
  live embeddings API.

## Out of scope

- File watching / triggering re-embeds (T-5).
- The MCP `search` tool itself (T-6).

## Verification commands

- `node mcp-server/lib/index-store.test.js`
- `node mcp-server/lib/similarity.test.js`
