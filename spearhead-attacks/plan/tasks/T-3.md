## Goal

Add `cssclasses` to `lib/knowledge-frontmatter.js`'s `LIST_FIELDS`, document the expanded `type` taxonomy, and ship an opt-in CSS snippet for Obsidian color-coding.

## Expected files

- `lib/knowledge-frontmatter.js`
- `lib/knowledge-frontmatter.test.js`
- `spearhead-knowledge/obsidian-css-snippet.css` (create)

## Depends on

none

## Acceptance criteria

- `LIST_FIELDS` includes `'cssclasses'` alongside the existing `'tags'`/`'related'` — parsed and serialized as an array, round-tripping exactly through `parse(serialize(x))` the same way `tags`/`related` already do.
- `type` values `code`, `decisions`, `design`, `architecture` are documented (in a code comment near `SCALAR_FIELDS`/`LIST_FIELDS`, or equivalent) as the four valid note categories — `design` is new (specified in A-1's original design as "opportunistic capture," never previously documented in code); no parser/validator code change is required beyond documentation, since `type` is already handled as an arbitrary scalar.
- `spearhead-knowledge/obsidian-css-snippet.css` exists, providing distinct visual styling (e.g. a colored left-border or background tint) for at least the four `cssclasses` values implied by the four note types (e.g. `kb-code`, `kb-decisions`, `kb-design`, `kb-architecture`), with a comment at the top of the file explaining it's opt-in — the user must manually copy it into their vault's `.obsidian/snippets/` and enable it in Obsidian's Appearance settings; this repo does not commit or reference any `.obsidian/` configuration.
- Tests cover: `cssclasses` round-trips through `parse(serialize(x))` with an array value; a note with no `cssclasses` field parses as `undefined`, not thrown/defaulted; malformed frontmatter's existing fallback (`{type: 'unknown'}`) is unaffected by the new field.

## Out of scope

- Any change to `hooks/knowledge-nudge.js`'s nudge text to mention `cssclasses` explicitly (out of scope for this attack's task breakdown — the frontmatter *capability* is what this task ships; whether/how nudge text references it is not required by any acceptance criterion).
- Bundling or committing any `.obsidian/` workspace/graph configuration — confirmed out of scope for the whole attack.
- `agents/guru.md` (T-1), `mcp-server/` deletion (T-5), `scripts/obsidian-graph.js` (T-6) — unrelated to this task.
