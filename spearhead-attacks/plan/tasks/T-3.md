## Goal

Pure modules for content-hashing (change detection) and calling the
embeddings API via `fetch`, with named errors on a missing key or failed
call.

## Expected files

- `mcp-server/lib/hash.js`
- `mcp-server/lib/hash.test.js`
- `mcp-server/lib/embeddings.js`
- `mcp-server/lib/embeddings.test.js`

## Depends on

T-1

## Acceptance criteria

- `hash.js` computes a stable `sha256` (`node:crypto`) of file content;
  identical content always produces the identical hash.
- `embeddings.js` reads `SPEARHEAD_EMBEDDINGS_API_KEY` (and optional
  `SPEARHEAD_EMBEDDINGS_ENDPOINT`) from the environment and calls the
  embeddings API via built-in `fetch` — no HTTP client dependency
  (PROBLEM.md #10, DESIGN.md open questions).
- Missing API key or a failed/erroring fetch call throws/rejects with a
  clear, named error (not a silent empty result) (PROBLEM.md acceptance
  criterion 10).
- Tests mock `fetch` (no live network calls in the test suite).

## Out of scope

- Index storage or similarity ranking (T-4).
- File watching (T-5).

## Verification commands

- `node mcp-server/lib/hash.test.js`
- `node mcp-server/lib/embeddings.test.js`
