## Goal

Document the second-brain feature for anyone installing the plugin, and
record the change in `CHANGELOG.md`.

## Expected files

- `README.md`
- `CHANGELOG.md`

## Depends on

T-6, T-8, T-9

## Acceptance criteria

- `README.md` gains a section describing the second-brain feature: the
  three knowledge sources, the `spearhead-knowledge/` layout, the `search`
  MCP tool, the `SPEARHEAD_EMBEDDINGS_API_KEY` requirement, and the
  opportunistic capture triggers (first-read, task-done).
- `CHANGELOG.md` gets a new version entry summarizing the feature,
  consistent with the existing entry format and style.

## Out of scope

- Any functional code changes — documentation only.

## Verification commands

- `node -e "require('fs').readFileSync('README.md','utf8')"` (sanity: file
  parses/reads without error)
