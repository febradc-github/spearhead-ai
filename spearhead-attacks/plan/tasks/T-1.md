## Goal

Bundle a minimal MCP server that both Claude Code and kimi-code load,
exposing a stub `search` tool, proving the core architectural assumption
(dual-runtime MCP server support) before anything else is built on top of it.

## Expected files

- `mcp-server/package.json`
- `mcp-server/package-lock.json`
- `mcp-server/server.js`
- `mcp-server/server.test.js`
- `.claude-plugin/plugin.json`
- `.kimi-plugin/plugin.json`
- `.gitignore`

## Depends on

none

## Acceptance criteria

- Both `.claude-plugin/plugin.json` and `.kimi-plugin/plugin.json` declare
  an `mcpServers` entry pointing at `mcp-server/server.js`, and remain valid
  JSON (PROBLEM.md #11: ships as part of the plugin, both manifests).
- `mcp-server/server.js` starts and registers a `search` tool via the MCP
  SDK (stub implementation is acceptable here; full logic lands in T-6).
- `mcp-server/node_modules/` is gitignored; `mcp-server/package-lock.json`
  is committed.

## Out of scope

- Real search/embeddings/index logic (T-3 through T-6).
- Any hook changes (T-8, T-9).
- Frontmatter parsing (T-2).

## Verification commands

- `cd mcp-server && npm install`
- `node mcp-server/server.test.js`
- `node -e "JSON.parse(require('fs').readFileSync('.claude-plugin/plugin.json','utf8'))"`
- `node -e "JSON.parse(require('fs').readFileSync('.kimi-plugin/plugin.json','utf8'))"`
