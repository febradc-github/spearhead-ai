## Goal

A dependency-free frontmatter parser/serializer for knowledge notes
(`type`, `tags`, `related`, `source`, `updated`), shared by `scripts/`,
`hooks/`, and `mcp-server/`.

## Expected files

- `lib/knowledge-frontmatter.js`
- `lib/knowledge-frontmatter.test.js`

## Depends on

none

## Acceptance criteria

- Parses `type`, `tags` (list), `related` (list of `[[wikilink]]` strings),
  `source`, `updated` from a note's leading `---` frontmatter block.
- Malformed or missing frontmatter falls back to `type: unknown` rather
  than throwing (DESIGN.md failure-mode handling).
- Serializes the same fields back to a valid frontmatter block
  (round-trips: parse(serialize(x)) === x for well-formed input).
- No npm dependencies (Node built-ins only), consistent with `scripts/` and
  `hooks/`.

## Out of scope

- MCP server logic, hashing, embeddings (T-1, T-3, T-4, T-5, T-6).
- Naming/collision logic (T-7).

## Verification commands

- `node lib/knowledge-frontmatter.test.js`
