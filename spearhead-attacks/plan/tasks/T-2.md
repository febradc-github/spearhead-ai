## Goal

Extend `hooks/knowledge-nudge.js`'s `handleRead` to detect source-content staleness via `source_hash` (T-1) instead of only checking note existence, and add wikilink-discipline guidance to all three nudge message sites.

## Expected files

- `hooks/knowledge-nudge.js`
- `hooks/knowledge-nudge.test.js`

## Depends on

T-1

## Acceptance criteria

- `handleRead` computes the current source file's content hash via `mcp-server/lib/hash.js`'s `hashContent` (imported directly, no new hashing logic) and derives a three-way state: no note → `new`; note exists with `source_hash` matching the current hash → `current`; note exists with `source_hash` missing or mismatched → `stale`.
- State `current`: never nudges, regardless of session (PROBLEM.md criterion 3, unchanged from today's existing-note behavior).
- State `stale`: nudges with refresh framing — names the note's existing path, asks for an in-place update plus a new `## Changelog` entry (not a duplicate note) (PROBLEM.md criterion 4).
- Both `new` and `stale` nudges go through the existing session-throttle check before firing.
- `shouldNudge`'s session-throttle schema changes from tracking an array of nudged paths to a map of path → last-nudged-hash (`nudged[relPath] === currentHash` replaces array-membership); a file that changes again after being nudged once naturally re-nudges, since its hash changes (PROBLEM.md criterion 5 — no double-nudge for the same stale state within a session/idle window). Old-format state files (array `nudged`) are treated as empty on load, never crash.
- Both `new`-note and `stale`-refresh nudge messages instruct the agent to set `source_hash` in the note's frontmatter.
- All three nudge message sites (`handleRead` new-note, `handleRead` refresh, `handleBash` task-done) include a line instructing the agent to use `[[wikilinks]]` only for genuinely related notes, never indiscriminately (PROBLEM.md criterion 6).
- `handleBash`'s detection logic (which task, which files, when it fires) is unchanged — only its message text gains the wikilink line.
- Hash computation and frontmatter parsing failures degrade gracefully (never crash the hook): unreadable source at read time → silent, no nudge; malformed note frontmatter → `source_hash` reads as `undefined`, which correctly falls into the `stale` branch.
- Tests (`spawnSync` fixtures, same pattern as existing) cover: matching-hash silence, missing/mismatched-hash refresh nudge, same-`(path, hash)` no-repeat throttling within a session, a changed file re-nudging after being nudged once, and wikilink-line presence in all three message call sites.

## Out of scope
