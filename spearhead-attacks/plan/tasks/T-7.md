## Goal

A deterministic, sole-authority script that computes a source file's
canonical, collision-safe knowledge-note path.

## Expected files

- `scripts/knowledge-path.js`
- `scripts/knowledge-path.test.js`

## Depends on

T-2

## Acceptance criteria

- Given a source file path, prints
  `spearhead-knowledge/code/<parent-folder>-<basename>.md`.
- If a note already exists at that computed slug with a *different*
  `source:` frontmatter value (via `lib/knowledge-frontmatter.js`),
  escalates one more parent level for the path being computed; existing
  notes are never renamed (PROBLEM.md acceptance criterion 5, DESIGN.md
  ADR-004).
- If a note already exists at the computed slug with the *same* `source:`
  value, returns that same path (it's a refresh, not a new note).
- Creates `spearhead-knowledge/code/` if it doesn't exist.
- Tests use a temp directory with fixture notes to exercise the
  no-collision, escalated-collision, and same-source-refresh cases.

## Out of scope

- Any MCP server logic (T-1, T-3–T-6).
- Actually writing doc content (that's the agent's job, nudged by T-9).

## Verification commands

- `node scripts/knowledge-path.test.js`
